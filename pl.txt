## Git 개념 정리

## 1. Git이란?
Git은 **분산 버전 관리 시스템(Distributed Version Control System)** 이다.  
소스 코드의 변경 이력을 기록하고 여러 개발자가 협업할 수 있도록 도와준다.

### 특징
- 버전 관리
- 협업 가능
- 변경 이력 추적
- 브랜치 기반 개발

---

# 2. Git 기본 구조

Git은 크게 3가지 영역으로 구성된다.


## 1) Working Directory
- 실제 작업하는 공간
- 파일을 수정하거나 생성하는 곳

## 2) Staging Area
- 커밋할 파일을 **임시로 저장하는 공간**
- `git add` 로 파일을 올림

## 3) Repository
- 커밋된 파일들이 저장되는 공간
- 프로젝트의 **버전 기록이 저장됨**
